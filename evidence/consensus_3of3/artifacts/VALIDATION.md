# Validation Record — Multi-Context Consensus 3-of-3

Reproducible evidence that `mcc_core.ConsensusVerifier` enforces a **3-of-3**
policy and **fails closed** under every adversarial input. Generated by
`harness.py` (deterministic); independently re-checked by
`verify_independent.py`.

- Policy: `threshold=3`, `veto_on_deny=true`, `on_fail=DENY`
- Operation: action `deploy_release`, payload `{"target": "cluster-prod-1", "environment": "prod"}`, actor `agent/ops`
- Evaluation time: `1780000000` (fixed); votes valid in `[1779999940, 1780003600)`
- Evaluator keys: derived from fixed seeds (deterministic); public keys in
  `artifacts/evaluators.json`

## Result: ALL PASS ✅

| scenario | kind | expected | actual | ok | description |
|---|---|---|---|---|---|
| `unanimous_3of3` | positive | ALLOW | ALLOW | ✅ | Three independent evaluators all sign ALLOW. |
| `below_threshold_2of3` | adversarial | DENY | DENY | ✅ | Only two ALLOW; the third ESCALATEs. |
| `veto_one_deny` | adversarial | DENY | DENY | ✅ | Two ALLOW, one trusted DENY vetoes. |
| `forged_untrusted_evaluator` | adversarial | DENY | DENY | ✅ | A rogue (untrusted) key signs the third ALLOW. |
| `tampered_vote` | adversarial | DENY | DENY | ✅ | A valid ALLOW vote is mutated after signing. |
| `duplicate_evaluator` | adversarial | DENY | DENY | ✅ | One evaluator casts three ALLOW ballots. |
| `wrong_action_binding` | adversarial | DENY | DENY | ✅ | Third vote is signed over a different action. |
| `wrong_payload_binding` | adversarial | DENY | DENY | ✅ | Third vote is signed over a different payload. |
| `wrong_actor_binding` | adversarial | DENY | DENY | ✅ | Third vote is signed for a different actor. |
| `expired_vote` | adversarial | DENY | DENY | ✅ | Third vote is outside its validity window. |
| `malformed_votes` | adversarial | DENY | DENY | ✅ | The votes payload is not a list. |

A "positive" scenario must reach consensus (ALLOW); every "adversarial" scenario
must **DENY** — no forged, untrusted, duplicated, mutated, mis-bound, or expired
vote can manufacture consensus, and any single trusted DENY vetoes.

## Reproduce

```bash
python evidence/consensus_3of3/harness.py            # regenerate artifacts
python evidence/consensus_3of3/verify_independent.py # independent re-verification
```

Ed25519 signing is deterministic, so regeneration yields byte-identical
artifacts; CI runs `harness.py --check` and fails on any drift. Integrity of
every artifact is pinned in `MANIFEST.sha256`.
